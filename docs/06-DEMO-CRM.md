## 1. 데모 시스템

### 데모 유형

| 유형 | 설명 | 대상 |
|------|------|------|
| 온라인 데모 | 화상 통화로 제품 시연 + Q&A (15~30분) | 기본 |
| 방문 데모 | 영업사원이 병원 방문 (hot 리드) | 요청 시 |
| 셀프 비디오 | 녹화된 데모 영상 제공 | 시간 안 맞을 때 |

### 데모 신청 폼 (공개 웹)

```
URL: www.madmedsales.com/demo

필수 입력:
- 병원명
- 원장님 성함
- 전화번호 (카카오톡 연동용)
- 이메일
- 희망 데모 유형 (온라인/방문/셀프비디오)
- 희망 일시 (날짜 + 시간 슬롯 선택)

선택 입력:
- 관심 분야 (리프팅/타이트닝/복합시술/기타)
- 현재 보유 RF 장비
- 궁금한 점
```

### API

```typescript
// routes/public.ts

/**
 * POST /api/public/demos/request
 * - 데모 신청 접수 (인증 불필요)
 * - 이메일로 lead 매칭 (없으면 새 lead + hospital 생성)
 * - demos 테이블 INSERT
 * - lead_activities 기록
 * - 관리자 알림 (카카오톡 + 이메일)
 * - 자동 확인 이메일 발송 (신청자에게)
 * - 카카오 알림톡 발송 (전화번호 있으면)
 */

// routes/demos.ts (admin API)

/**
 * GET /api/demos              - 데모 목록 (필터: status, date)
 * GET /api/demos/:id          - 데모 상세
 * PUT /api/demos/:id/confirm  - 데모 확인 (일시 확정)
 * PUT /api/demos/:id/prepare  - AI 자동 자료 생성 트리거
 * PUT /api/demos/:id/complete - 데모 완료 처리
 * PUT /api/demos/:id/cancel   - 데모 취소
 * 
 * POST /api/public/demos/:id/evaluate - 원장 평가 제출 (공개)
 */
```

### 데모 자동 준비 자료 (AI 생성)

```typescript
/**
 * PUT /api/demos/:id/prepare 호출 시:
 * 
 * Claude API로 3가지 자료 자동 생성:
 * 
 * 1. prep_scoring_summary: 병원 분석 요약
 *    "이 병원은 강남에 위치한 피부과로, 울쎄라는 보유하고 있지만
 *     RF 장비가 없어 리프팅 풀코스 구성에 공백이 있습니다.
 *     상권 내 RF 보유율은 20%로, TORR RF 도입 시 차별화 효과가 큽니다."
 * 
 * 2. prep_roi_simulation: ROI 시뮬레이션
 *    {
 *      "monthly_sessions": 40,           // 월 예상 시술 건수
 *      "avg_session_price": 350000,      // 건당 평균 가격
 *      "monthly_revenue": 14000000,      // 월 예상 매출
 *      "annual_revenue": 168000000,      // 연 예상 매출
 *      "equipment_cost": 27000000,       // 장비 투자비
 *      "roi_months": 2,                  // 투자 회수 기간 (개월)
 *      "assumptions": "강남 피부과 평균 기준, 월 40건 보수 추정"
 *    }
 * 
 * 3. prep_combo_suggestion: 시술 조합 제안
 *    "원장님의 울쎄라 + TORR RF 조합으로:
 *     - 딥 리프팅(울쎄라) + 표면 타이트닝(TORR RF) 풀코스
 *     - 유지 시술로 TORR RF 단독 프로그램
 *     - 월 패키지 상품화 가능"
 */
```

### 데모 후 원장 평가 폼

```
URL: www.madmedsales.com/demo/{demo_id}/evaluate?token={token}

입력:
- 만족도 (1~5 별점)
- 구매 의향: 즉시 구매 / 검토 중 / 보류 / 관심 없음
- 선호 결제 방식: 일시불 / 할부 / 렌탈 / 캐피탈
- 추가 궁금한 점 (자유 텍스트)
- 한줄 피드백 (자유 텍스트)
```

### 데모 후 자동 후속 조치

```typescript
/**
 * 평가 결과 기반 자동 분기:
 */

if (evaluation.purchase_intent === 'immediate') {
  // 즉시 구매 → 견적서 + 결제 안내
  await sendQuotationEmail(lead);
  await sendAlimtalk(lead, 'QUOTATION_SEND');
  await updateLead(lead.id, { stage: 'proposal' });
  await notifyAdmin('🔥 즉시구매 의향! ' + hospital.name);
}

if (evaluation.purchase_intent === 'considering') {
  // 검토 중 → 추가 자료 + 1주 후 팔로업
  await sendAdditionalMaterials(lead);
  await scheduleFollowup(lead, 7);  // 7일 후
  await updateLead(lead.id, { stage: 'negotiation' });
}

if (evaluation.purchase_intent === 'hold') {
  // 보류 → 감사 메시지 + 3개월 후 재접근
  await sendThankYouEmail(lead);
  await scheduleFollowup(lead, 90);  // 90일 후
  await updateLead(lead.id, { stage: 'nurturing' });
}

if (evaluation.purchase_intent === 'no_interest') {
  // 관심 없음 → 마무리
  await sendThankYouEmail(lead);
  await updateLead(lead.id, { 
    stage: 'closed_lost', 
    lost_reason: 'no_interest_after_demo' 
  });
}
```

## 3. 영업팀 연결 구조

### 리드 카드 (영업팀에 전달하는 정보)

```
영업팀에 리드를 넘길 때 제공하는 정보 패키지:

1. 병원 기본 정보 (이름, 주소, 전화, 원장)
2. 스코어링 결과 + AI 분석 메모
3. 전체 접촉 이력 (이메일/카카오톡 대화 내용)
4. 원장 반응 패턴 (AI 분석)
   "이 원장님은 가격보다 임상 데이터에 관심이 많습니다.
    3번 이메일에서 케이스 스터디를 클릭했고,
    데모에서 'ROI가 중요하다'고 언급했습니다."
5. 데모 평가 결과 (있으면)
6. 추천 접근법 (AI 추천)

전달 방식:
- admin 대시보드에서 "영업팀 배정" 버튼
- 리드 카드 PDF 자동 생성 (또는 카카오톡으로 요약 전달)
```

### 수수료 구조

```
TORR RF 가격: 25~28M (중간값 26.5M 기준)

가격 구성:
- 제조원가: 10M
- 본사 마진: 10M
- 판매 수수료: 5~8M (중간값 6.5M)

수수료 분배:
- MADMEDSALES가 직접 클로징: 수수료 100% → MADMEDSALES
- 영업팀 연결: 수수료 50:50 (기본, 딜러별 조정 가능)
  → MADMEDSALES: 3.25M
  → 영업팀/딜러: 3.25M
```

```
□ 데모 신청 → 접수 → 관리자 알림
□ 데모 확인 → 알림톡 발송
□ 데모 완료 → 평가 폼 발송
□ 평가 제출 → 자동 후속 조치
□ 수신거부 → 즉시 차단
□ CRM 대시보드 데이터 정확성
```

## 7. KPI 목표 (런칭 후 1개월)

```
이메일:
- 도달률: 95%+ (Resend 기본값)
- 오픈율: 30%+ (업계 평균 15~20%)
```
