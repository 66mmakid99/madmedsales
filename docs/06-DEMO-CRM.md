# Phase 5: 데모 시스템 + CRM 대시보드 + 런칭 (Week 11~12)

## 이 Phase의 목표

리드가 데모까지 도달했을 때의 프로세스를 완성하고, 전체를 관리할 CRM 대시보드를 구축하여 시스템 런칭.

## 핵심 개념: 제품별 데모 + 통합 CRM

```
같은 병원이라도 제품에 따라 데모가 다름:

TORR RF → 방문 데모 or 온라인 데모 (30분, ROI 시뮬레이션 포함)
소모품 → 데모 불필요 (샘플 발송 or 바로 견적)
관리장비 → 자체 영상 or 15분 온라인 데모

CRM 대시보드에서는:
→ 제품별 파이프라인 (필터링)
→ 전체 병원 통합 뷰 (한 병원의 여러 제품 리드를 한눈에)
```

## 선행 조건

- Phase 4 완료 (팔로업 + 카카오 + 관심도 시스템 작동)
- HOT 리드 일정 수 확보

## 1. 데모 프로세스

### 제품 유형별 데모

| 제품 유형 | 데모 타입 | 소요 시간 | 준비물 |
|----------|----------|----------|--------|
| 고가 장비 (TORR RF 등) | 방문 or 온라인 | 30분 | ROI 시뮬레이션, 시술 조합 제안, 경쟁 분석 |
| 중가 장비 | 온라인 or 셀프 영상 | 15분 | 제품 비교 자료, 사례 |
| 소모품 | 데모 불필요 | - | 견적서, 샘플 |

### AI 데모 준비 자료 자동 생성

```typescript
const DEMO_PREP_PROMPT = `
당신은 의료기기 데모 전문가입니다.
아래 정보를 바탕으로 이 병원 맞춤 데모 자료를 준비하세요.

## 제품
{{product_info}}

## 병원 분석
- 프로파일: {{profile_summary}}
- 매칭 분석: {{match_analysis}}
- 이메일 소통 이력: {{email_history_summary}}
- 관심 포인트: {{interest_signals}}

## 생성할 자료 (JSON)
{
  "prep_summary": "데모 전 브리핑 (영업 담당자용, 5줄)",
  "roi_simulation": {
    "monthly_revenue_increase": "예상 월 매출 증가분",
    "payback_period": "투자 회수 기간",
    "assumptions": ["가정 1", "가정 2"]
  },
  "selling_points": ["이 병원에 맞는 핵심 포인트 3개"],
  "objection_handling": [
    {"objection": "예상 반론", "response": "대응 방안"}
  ],
  "recommended_demo_flow": ["데모 진행 순서 (단계별)"]
}
`;
```

### 데모 후 평가

```typescript
/**
 * 데모 완료 후 admin에서 입력:
 * 
 * 1. satisfaction_score (1~5)
 * 2. purchase_intent: immediate / considering / hold / no_interest
 * 3. preferred_payment: lump_sum / installment / rental / capital
 * 4. additional_questions: 추가 질문
 * 5. feedback: 자유 피드백
 * 
 * 자동 후속 조치:
 * - immediate → 견적서 자동 생성 + 계약 프로세스
 * - considering → 1주일 후 팔로업 이메일 예약
 * - hold → 1개월 후 리마인드
 * - no_interest → nurturing으로 이동 + 3개월 재접근
 */
```

## 2. CRM 대시보드 (admin 앱)

