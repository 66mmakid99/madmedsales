# Phase 6: 온라인 결제 시스템 (추후 개발)

## 이 Phase의 목표

온라인으로 TORR RF 구매 결제까지 완료할 수 있는 시스템. 전체 시스템이 안정적으로 운영된 후 개발.

## 선행 조건

- Phase 0~5 완료 + 안정 운영 (최소 1~2개월)
- 실제 계약 2~3건 이상 발생 (수동 결제)
- 결제 방식별 수요 파악 완료

## 개발 시기

- 런칭 후 2~3개월 뒤 (운영 데이터 축적 후)
- 그 전에는 수동 결제 (계좌이체/대면 계약)

---

## 1. 결제 옵션

| 방식 | 금액 | 비고 |
|------|------|------|
| 일시불 (카드) | 25~28M | PG사 수수료 2~3% |
| 일시불 (계좌이체) | 25~28M | 수수료 없음 |
| 12개월 무이자 할부 | 월 2.08~2.33M | 카드사 제휴 필요 |
| 렌탈 | 월 TBD | 렌탈사 파트너십 |
| 캐피탈 매칭 | 의료기기 전문 캐피탈 | 캐피탈사 연결 |

---

## 2. PG 연동 (추후 검토)

```
후보:
- 토스페이먼츠: 의료기기 고가 결제 가능 여부 확인 필요
- NHN KCP: 고가 상품 경험 많음
- 이니시스: 가장 범용적

고가 결제 특이사항:
- 1회 결제 한도: 카드사별 다름 (보통 500만원~5000만원)
- 사업자 간 거래 (B2B): 세금계산서 발행 필요
- 계약금 + 잔금 분할 결제 가능성
```

---

## 3. 견적서 자동 생성

```
데모 평가에서 purchase_intent = 'immediate'일 때:

1. 견적서 PDF 자동 생성
   - 병원명, 일자
   - TORR RF 제품 사양
   - 가격 (협의된 금액)
   - 결제 옵션 안내
   - 유효기간 (30일)
   
2. 이메일 + 카카오톡으로 발송

3. 견적서 열람 추적 → 관리자 알림
```

---

## 4. 결제 플로우 (미래)

```
1. 견적서 확인 → "결제하기" 버튼
2. 결제 방식 선택 (일시불/할부/렌탈/캐피탈)
3. PG 결제 진행 (or 계좌이체 안내)
4. 결제 완료 → commissions 자동 생성
5. 세금계산서 발행
6. 배송/설치 일정 안내
```

---

## 현재 단계에서의 대안 (수동 처리)

```
런칭 초기에는:
1. 데모 후 구매 의향 확인
2. 견적서를 수동 작성 (또는 AI 자동 생성)
3. 이메일/카카오톡으로 견적서 전달
4. 계좌이체 or 대면 계약
5. commissions 테이블에 수동 입력

→ 결제 건수가 월 5건 이상 되면 자동화 시스템 개발
```
